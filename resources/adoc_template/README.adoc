== About the Document

The source text of this document is written in the https://asciidoc.org[AsciiDoc^] markup language.

=== Guideline

This document should follow the link:README-guideline.adoc[Writing Guideline].

=== Export

. To export the document to the supported output formats, install the following tools:
.. https://asciidoctor.org/#installation[Asciidoctor^]
.. https://docs.asciidoctor.org/pdf-converter/latest/install/[Asciidoctor PDF^]
.. https://pandoc.org/installing.html[Pandoc^]
.. https://www.python.org/downloads/[Python 3^]

. To export to HTML:
+
[source,bash,subs="attributes+"]
----
asciidoctor --attribute allow-uri-read {docname}{docfilesuffix}
----

. To export to PDF:
+
[source,bash,subs="attributes+"]
----
asciidoctor-pdf --attribute allow-uri-read --require ./docs/lib/common-style.rb {docname}{docfilesuffix}
----

. To export to DOCX - Asciidoctor converts the document to DocBook XML, which Pandoc then converts to DOCX:
+
[source,bash,subs="attributes+"]
----
asciidoctor --backend docbook --attribute allow-uri-read --out-file - {docname}{docfilesuffix} | pandoc --from docbook --to docx --reference-doc={sourcedir}/docs/themes/reference.docx --output {docname}.docx && python3 {sourcedir}/docs/lib/apply_docx_formatting.py {docname}.docx
----

=== Linting

==== General Rules

. The document is linted by `link:docs/lib/adoc_lint.py[]`, a single command that runs two engines and merges their findings into one report. The two engines are Vale and a structural linter.
. The first engine, https://vale.sh[Vale^], checks prose and typography. Vale is configured in `link:.vale.ini[]` and enforces the following style guides:
.. native https://vale.sh/docs/styles#vale[Vale style^], with https://github.com/vale-cli/en_US-web/tree/master[English dictionary^] included,
.. a curated `English` style, inspired by the https://developers.google.com/style[Google Developer Documentation Style Guide^], the https://vale.sh/hub/jdkato/write-good/[write-good^] linter, and https://github.com/amperser/proselint/[proselint^],
.. language-neutral typography rules, applied to every language.
. The second engine, the structural linter, checks the markup, document structure, and ASCII diagrams that Vale can't see, defined in `link:docs/lib/adoc_lint.py[]`.

==== Usage

. Install Vale as described in the https://vale.sh/docs/install/[official documentation^]. The linter requires Vale and exits with an error when Vale is missing. The structural checks need only Python 3, which the DOCX export above already requires.

. Lint the document:
+
[source,bash,subs="attributes+"]
----
python3 docs/lib/adoc_lint.py {docname}{docfilesuffix}
----
