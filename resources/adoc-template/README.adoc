== About the Document

This document uses https://asciidoc.org[AsciiDoc^] markup language and resides in the project repository.

=== Export
. To export the document to the supported output formats, install the following tools:
.. https://asciidoctor.org/#installation[Asciidoctor^]
.. https://docs.asciidoctor.org/pdf-converter/latest/install/[Asciidoctor PDF^]
.. https://pandoc.org/installing.html[Pandoc^]

. To export to HTML:
+
[source,bash,subs="attributes+"]
----
asciidoctor --attribute allow-uri-read {docname}{docfilesuffix}
----

. To export to PDF:
+
[source,bash,subs="attributes+"]
----
asciidoctor-pdf --attribute allow-uri-read --require ./docs/lib/auto-list-style.rb {docname}{docfilesuffix}
----

. To export to DOCX - Asciidoctor converts the document to DocBook XML, which Pandoc then converts to DOCX:
+
[source,bash,subs="attributes+"]
----
asciidoctor --backend docbook --attribute allow-uri-read --out-file - {docname}{docfilesuffix} | pandoc --from docbook --to docx --output {docname}{docfilesuffix}
----

=== Linting

==== General Rules
. https://vale.sh[Vale^] lints the document.
. Vale enforces the following style guides:
.. Native https://vale.sh/docs/styles#vale[Vale style^], with https://github.com/vale-cli/en_US-web/tree/master[English dictionary] included.
.. modified https://developers.google.com/style[Google Developer Documentation Style Guide^] via the https://github.com/vale-cli/Google[Google package^]
.. modified https://vale.sh/hub/jdkato/write-good/[write-good linter^] via the https://github.com/vale-cli/write-good[write-good package^]
.. https://github.com/amperser/proselint/[proselint^] via the https://github.com/vale-cli/proselint[proselint package^]
.. custom project rules
. Vale configuration is in `link:.vale.ini[]`.

==== Usage
. Install Vale as described in the https://vale.sh/docs/install/[official documentation].

. Lint the document:
+
[source,bash,subs="attributes+"]
----
vale {docname}{docfilesuffix}
----
